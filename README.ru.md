# School Mate

# 📚 Школьная Информационная Система

Шаблон веб-приложения для частных школ, образовательных центров и кружков, разработанный на Spring Boot.

## Обзор
Проект направлен на автоматизацию управления учебным процессом, взаимодействия с клиентами (родителями и учениками), а также учёта административных и финансовых процессов. Это базовая заготовка, включающая зависимости для веб-разработки, JPA, безопасности, Thymeleaf, Liquibase и PostgreSQL, чтобы обеспечить удобную настройку для распределённой команды.

### Основные модули
#### 🔒 Авторизация и роли
- Аутентификация пользователей по ролям: Администрация, Сотрудники, Клиенты (Ученики/Родители).
- Настраиваемая система прав доступа.

#### 🏛️ Администрация
- Управление пользователями (сотрудники, ученики, родители).
- Расписание занятий и назначение учителей.
- Электронный дневник (ведение классов, домашних заданий, предметов и оценок).
- Управление аудиториями и ресурсами.
- 📦 Складской учёт (учёт оборудования и других ценностей).
- 💰 Финансовый учёт:
  - Учёт поступлений и возвратов от клиентов.
  - Зарплаты сотрудников и расчёты.
  - Прочие платежи.
- 📊 Отчётность по ключевым показателям (финансы, посещаемость, успеваемость).
- 📤 Импорт/экспорт данных (Excel, CSV).

#### 👨‍🏫 Сотрудники (Учителя, Персонал)
- Просмотр и редактирование электронного дневника.
- Ведение расписания, оценок и домашних заданий.
- Учёт посещаемости.
- Индивидуальные комментарии ученикам и родителям.
- Портфолио ученика (достижения, заметки).
- Ответы на вопросы родителей.

#### 👨‍👩‍👧 Клиенты (Ученики и Родители)
- Просмотр дневника, оценок, расписания.
- Личный кабинет (домашние задания, задолженности).
- Возможность комментировать и задавать вопросы преподавателям.
- История занятий и успеваемости.
- Запись на дополнительные курсы и мероприятия.
- Электронные документы (квитанции, справки, договоры).

### Перспективы развития
- Поддержка мобильной версии (адаптивный дизайн).
- Поддержка международных языков (i18n).
- REST API для внешних интеграций.
- Интеграция с мессенджерами (например, Telegram-бот для уведомлений).
- Облачное хранилище для файлов и документов.

## Технические детали
- **Backend**: Java, Spring Boot
- **Frontend**: Thymeleaf (с использованием Bootstrap)
- **База данных**: PostgreSQL (или любая другая по выбору)
- **Безопасность**: Spring Security (роль-ориентированная авторизация)
- **Тестирование**: JUnit, Mockito
- **Документация API**: Springdoc OpenAPI с Swagger UI (`/swagger-ui.html`) для документирования и тестирования REST API.

## Требования
- **Java 17**
- **Maven 3.8+**
- **Docker и Docker Compose** (опционально, для контейнеризированной настройки)
- **PostgreSQL** (опционально, для работы с базой данных)
- IDE (IntelliJ IDEA, Eclipse, VS Code и т.д.)

## Установка через Maven

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/fedeyak/school_mate.git

2. Перейдите в директорию проекта:
   cd school_mate

3. Соберите проект:
   mvn clean install

4. Запустите приложение:
   mvn spring-boot:run

5. Откройте Swagger UI для просмотра и тестирования API:
    - URL: `http://localhost:8080/swagger-ui.html`
    - OpenAPI JSON: `http://localhost:8080/api-docs`
   
Приложение запускается на http://localhost:8080. 
По умолчанию Liquibase и JPA отключены 
(spring.liquibase.enabled=false, spring.jpa.hibernate.ddl-auto=none), 
поэтому база данных не требуется.

## Установка через Docker

1. Убедитесь, что Docker и Docker Compose установлены:
   docker --version
   docker-compose --version

2. Соберите и запустите приложение и базу данных PostgreSQL:
   docker-compose up --build

3. Откройте приложение по адресу http://localhost:8080.

4. (Опционально) Подключитесь к базе данных PostgreSQL на localhost:5432 с параметрами:
- База данных: school_mate
- Имя пользователя: postgres
- Пароль: postgres

5. Остановите контейнеры:
   docker-compose down

## Настройка базы данных

Проект настроен для работы с PostgreSQL и Liquibase, 
но база данных необязательна на начальном этапе разработки.
Для включения базы данных:

1. 1. Настройте `application.yml` (или создайте `application-local.yml` для локальных настроек):
   spring:
       datasource:
           url: jdbc:postgresql://localhost:5432/school_mate
           username: your_username
           password: your_password
           driver-class-name: org.postgresql.Driver
       liquibase:
           enabled: true
           change-log: classpath:db/changelog/db.changelog-master.yaml
       jpa:
         hibernate:
            ddl-auto: none

2. Создайте файл changelog для Liquibase 
(например, src/main/resources/db/changelog/db.changelog-master.yaml) для добавления миграций.

3. Убедитесь, что PostgreSQL запущен локально, или используйте Docker-настройку выше.

## Устранение неполадок

- Проблемы с Maven: Выполните mvn clean install для устранения ошибок зависимостей.
- Проблемы с Docker: Убедитесь, что Docker Desktop запущен, и повторите docker-compose up --build.
- Ошибки базы данных: Проверьте настройки в application.yml и доступность PostgreSQL.

- Для дополнительной помощи обратитесь к команде разработки или проверьте issues в репозитории.
